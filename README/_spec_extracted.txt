0101: VENDOR ABSTRACTION PRINCIPLE
0102: Every external dependency sits behind an adapter ABC in Python.
0103: Config file selects which adapter loads at startup.
0104: Swapping vendors = writing one new file in adapters/{layer}/
0105: Adapter locations: adapters/search/ | adapters/llm/ | adapters/graph/ | adapters/queue/ | adapters/telemetry/
0106: OpenTelemetry is the one exception Ś already the open standard, no adapter needed.
0107: Lock-in is real but contained to adapter files. Core business logic is entirely vendor-neutral.
0108: TELEMETRY PRINCIPLE Ś CRITICAL
0109: AI applications are instrumented, not aware.
0110: Connected applications do not know they are being governed.
0111: They do not calculate KPIs. They do not call the governance layer for analytics.
0112: They simply run and emit standard OTEL telemetry as a side effect of normal operation.
0113: The governance layer is the active party Ś it pulls, calculates, and interprets everything.
